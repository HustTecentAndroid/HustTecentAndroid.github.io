对KMP算法的理解

本文总共2552个字 | 您是第位看到它们的小伙伴

引入

KMP算法是一种比较高效字符串匹配算法，用于在母串中查找子串，常用于文本搜索等场合，比较实用。但比较难以理解，特别是在严蔚敏版的《数据结构》教材中讲的更是晦涩难懂，所以各方查找资料理解，也总算有了一点自己的理解。

算法思想

传统的字符串查找中，通常都是一个个字符匹配，发现不匹配时则将子串向后移动一位，再接着与目标串匹配，如此循环往复。他的原理简单易懂，实现也是十分简单，但查找效率十分低下。而KMP算法的创新思想在于，当子串与目标串不匹配时，其实你已经知道了前面已经匹配成功的那一部分字符串（包括子串和目标串），而它设法利用这个已知信息，不把“搜索位置”移回已经比较过的位置，继续把它后移，从而大大提高效率。

得到每次右移位数

通过上述逐步分析可以很清楚的看出，在有字符不匹配时，确定每次右移的位数时，其实主要是寻找到当前已匹配到的字符串中首尾由多少个相同的字符。我们假设子串为Sub，目标串为Target，即当Sub[j]不匹配时，即Sub[j]!=Target[k]时，我们可以知道Sub[0...j-1]与Targer[k-j...k-1]是相等的，我们只需要找到已匹配到的字符串中的首尾有多少个字符相同，即Sub[0...t-1]与Sub[j-t...j-1]相等的最大的t的值，确定了这个t，我们就可以利用Sub[j-t...j-1] = Target[k-1-((j-1)-(j-t))...k-1],即Sub[j-t...j-1] = Target[k-t...k-1],易得Sub[0...t-1] = Target[k-t...j-1]，即可以将子串的Sub[0]与Target[k-t]比较，而原来Sub[0] = Target[k-j]，所以右移位数为(k-t)-(k-j),即为j-t,即得出KMP算法的公式: 移动位数 = 已匹配的字符数 - 对应的部分匹配值

确定部分匹配值

由上述分析可知，对应的部分匹配值即为已匹配的字符串中首尾的最大相同字符数。在子串中，对于每一个Sub[j]，均存在一个确定的部分匹配值，设为next[j]，关键是求解next[j]。对于next[j]，我们容易得到有如下性质:

若已知next[0...j],则对于一个next[j],有Sub[0...(next[j]-1)] = Sub[(j-next[j])...j-1]

  1. next[0] = -1
  2. 若Sub[j] == Sub[next[j]],则有next[j+1] = next[j] + 1
  3. 若Sub[j] != Sub[next[j]],设next[j] = k,若此时有Sub[next[k]] == Sub[j]，则有next[j+1] = next[k] + 1，否则，则可以继续递归k = next[k],而后重复此过程。
